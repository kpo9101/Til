## <strong style="font-size: 36px; color: rgb(255, 255, 255);">2. 학습 내용</strong>
```
베열
: 자료구조 중 가장 기초
사용할 메모리 크기를 고정해서 선언
선언 후에는 절대 크기 변경 불가
정적인 자료구조
선언된 메모리는 연속적으로 할당
배열의 index는 항상 0부터 시작
```

```
함수
: 코드뭉치
반복되는 코드 뭉치가 있다면 함수로 만들어서 재사용성 높일 수 있다
입력 혹은 출력이 있는 경우 대부분 다 함수
```

```
[중요 샘플 코드]
반환 자료형 함수명 (매개변수자료형 매개변수명)
{
    return 반환값;
}

int main(void)
{
    int num = 10; // 자료형 변수명 = 값;

    함수명(인자값); // 함수 호출(call)

    return 0;
}
```

```
스택 메모리
: 함수 호출에 할당될 메모리
지역변수들이 저장되는 공간
스택 메모리는 스택 포인터와 베이스 포인터, 스택 프레임들로 구성
# 스택 프레임
: 함수가 호출되면 해당 함수가 사용할 메모리 크기만큼 공간이 확보된다
 해당 함수를 위해 확보된 메모리 공간
 나중에 함수가 모두 수행된 후 해당 스택 프레임은 다시 반환된다
 
 # 스택 포인터
 : 현재 스택 프레임의 Top를 가르킨다
   push 명령어나 pop 명령어의 피연산자

# 베이스 포인터
: 현재 스텍 프레임의 시작 주소 
```

```
전방 선언
: 함수의 원형만 따서 파일 스코프 상단
  함수의 정의는 파일 스코프의 하단에 위치
```

```
변수의 종류
: 지역변수/전역변수
여기에 static, const, extren 같은 키워드가 붙으면 뉘앙스가 달라진다

# 지역변수
: 블럭 스코프 내에 선언된 변수. 스택 메모리에 저장
함수가 종료되면 스택 프레임이 반환되면서 더이상 접근 불가

# 정적 지역 변수
: 지역 변수 앞에 static 키워드가 붙으면 데이터 섹션에 저장
함수 종료시 접근 불가한 스택메모리에 저장 x
함수가 종료되어도 값이 유

# 전역 변수
: 파일 스코프에 선언된 변수. 데이터 섹션에 저장

# 정적 전역 변수
: 전역 변수 앞 static 키워드가 붙으면 해당 변수는 해당 파일 내에서만 접근 가능

# const 변수
const 키워드가 붙은 변수. 초기화 이후에 값 변경 불가. 초기화 강제
```

```
포인터
: 매모리 주소를 저장하는 변수
```

```
[중요 샘플 코드]
자료형 변수명 = 값;

자료형* 변수명 = 메모리주소값; // 자료형 쪽에 *을 붙인다. 헷갈리지 않기 위함 
```

```
주소 연산자 &
피연산자의 메모리 주소를 반환하는 연산자
```

```
메모리 주소를 얻는 2가지 방법
1. 주소 연산자
2. 배열의 이름
```

```
역참조 연산자 *
피연산자로 포인터를 받아서, 해당 메모리 주소에 저장된 값을 읽거나 값을 수정할 때 사용하는 연산자.
```

```
[좋은 습관] 메모리 그리기
: 포인터 관련 예제를 풀 경우 "메모리 그리기"
 사각형을 그리고 변수를 적고 각 변수에 메모리 주소를 적는다
 그 상태로 한 줄씩 "천천히 읽기"
```

```
[중요 샘플 코드]
// Main.c

#include <stdio.h>

int main(void)
{
	int X = 10;
	int* PtrToX = &X;

	printf("%d\n", *PtrToX);

	*PtrToX = 100;

	printf("%d\n", *PtrToX);

	return 0;
}
```

```
NULL 포인터
#define NULL((void*)0)
아무것도 가르키지 않는 포인터
NULL 포인터 사용처
1. 포인터의 초기화
2. 포인터가 더이상 사용중이지 않음을 알리고 싶을 때
3. 포인터가 유효한 메모리 주소를 저장하고 있는지 확인할 때
```

```
void 포인터
: 범용 포인터
void* 변수명 = 메모리 주소 값;

 void 포인터 주의점 
 1. void*에 역참조 연산 불가능 -> 해당 메모리 주소부터 몇 바이트만큼 읽어야 내 데이터인지 모르기 때문
 2. void*에 정수를 더하거나 빼는 연산은 불가능 -> 해당 메모리 주소부터 몇 바이트만큼 더하거나 빼야하는지 모르기 때문
```
