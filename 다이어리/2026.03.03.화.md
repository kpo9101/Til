### 스택 메모리와 힙 메모리가 나뉜 이유
스택 메모리 단점


      1. 스택 메모리에 저장되는 지역 변수(배열 등)의 크기는 컴파일 타임에 고정된다. 런타임에 더 필요해진다 해도 크기를 늘릴 수 없습니다.
      2. 함수가 종료되면 해당 스택 프레임에 더이상 접근 불가 -> 지역변수의 수명은 함수의 수명과 같이한다.
        더 오래 보존하려면 전역변수 혹은 정적변수로 선언해야 한다.
          BUT 이 변수들은 극단적이다. 프로그램의 수명과 같이한다.

     -> 그렇기 때문에 런타임 시 프로그래머가 원하는 만큼(1번 만족) 원하는 때에 생성 및 삭제(2번 만족) 가능한 메모리가 필요하다

### 힙 메모리
    프로그래머의 메모리 할당과 해제를 통해 관리되는 동적 할당 영역이다.

장점 : 

      1. 프로그래머가 원하는 만큼, 원하는 때 할당 및 반납이 가능하다



단점 : 

      1. 스택 메모리에 비교해 할당/해제 속도가 느리다. 힙 메모리는 할당 받아오려면 사용중이지 않은 메모리이면서 크기가 맞는지 체크 후 제공된다.
         또한 메모리 공간에 구멍(메모리 단편화)가 생길 수도 있어서 효율적인 메모리 관리가 어렵다

      2. 프로그래머가 직접 메모리 할당 및 해제해야한다. 할당만 하고 해제는 안하는 실수를 할 수 있다.   

### 동적 메모리 관련 함수

    할당: malloc()
    재할당: realloc()
    해제: free()

### void* malloc(size_t size)
     memory allocation의 약자. stdlib.h 헤더파일에 선언되어 있다
     size 바이트 만큼의 메모리를 반환
     할당 실패시 NULL을 반환

### void free(void* ptr)
    동적 할당 받은 메모리를 해제하는 함수 -> 메모리 할당 함수를 통해서 얻은 메모리 주소만 해제 가능 

### [좋은 습관] malloc() 작성 후 바로 free() 작성
    반납하지 않으면 메모리 누수가 발생하는 문제가 생겨 무슨 일이 있어도 free()부터 바로 작성하기

### 동적 할당 받은 메모리 시작 주소를 연산에 사용하면?
    최초에 받아온 시작 주소를 잃어버린다. 그러면 다른 메모리 주소를 free() 함수의 인자로 보낼지 모른다.
    사본을 만들어서 포인트 연산에 사용해야 한다.

### [심화] void* realloc(void* ptr, size_t new_size)
     이미 동적할당 받은 메모리 시작 주소 ptr을 new_size 바이트로 재할당 해주는 함수
     새로운 크기가 허용하는 한 기존 데이텅를 그대로 유지

### 동적 할당 메모리 소유권
    해당 메모리 공간의 주인이 누구인지는 아주 중요한 문제다
    왜냐면 받으시 책임 지고 해제해야 하기 때문이다.
    또 주인이 아닌데 마음대로 해제해도 문제다.
    알아서 주인이 직접 처리하는거다


### 구조체(Structure)
    필요한 여러 자료형의 변수들을 한데 묵어서 하나의 자료형처럼 만들 수 있다.

    struct 구조체명
    {
        자료형 변수명;

    }; // 세미콜론(;) 해야한다
    
### [참고] 구조체와 객체지향 프로그래밍
    나만의 자료향으로 물체(객체)를 정의하고, 동적할당을 이용해서 
    물체를 생성 후 원하는 때에 해당 물체를 제거하는 것은
    객체지향 프로그래밍과 유사하다.


### 구조체도 기본 자료형으로 이용하려면?
    typedef : type defunution의 약자. 이미 정의되어 있는 자료형에 새로운 별명을 지어준다

### size_t 자료형도 사실 typedef unsigned long long size_t

### 구조체 멤버 접근 연산자 ->
    구조체 포인터에 역참조 연산자와 . 연산자를 합친 연산자가 -> 연산자.
    1순위 연산자. 괄호 안쳐도 됨

### 구조체에 멤버 변수만 선언 할 수 있는건 아니다.
    객체지향 프로그래밍으로 넘어가면서 구조체에는 멤버 함수도 선언 가능하다

### 구조체와 클래스가 완전히 똑같은 것은 아니다.
    클래스는 접근 제한 지정자와 상속 등 다양한 기능들이 추가적으로 지원된다

### 열거형 (Enummeration)
    정수에 별명을 붙여서 소스코드를 이해하기 쉽게 해준다

### 열거형도 typedef 가능하다
    
### 얕은 복사(Shallow Copy)
    실제 데이터가 아닌 데이터가 들어있는 메모리 주소만 복사되는 것
    두 변수가 다른 이름을 가지지만, 같은 메모리 주소를 복사해서 가지고 있기 때문 실제 데이터는 하나만 존재

### 깊은 복사(Deep Copy)
    서로 다른 메모리 주소를 가지면서, 해당 메모리 주소에 적힌 값은 같은 상태
    값은 같지만, 실제 데이터가 두 개 존재하는 상태
