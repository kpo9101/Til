## 회전하는 액터
    회전하는 액터를 만들기 위해서 블루프린트 클래스를 만들어보겠습니다. 
    콘텐츠 드로어에서 블루프린트 클래스를 생성해주세요. 이름은 RotationPlatform
![alt text](image-42.png)

    **‘Event Tick’** 과 **‘Add Actor Local Rotation’** 노드를 사용하면 금방입니다.
    **‘Platform Velocity’** 변수에는 회전 속도가 있습니다. Delta Seconds와 곱하여서 일정한 속도로 회전할 수 있도록 합니다.
![alt text](image-43.png)

    기본 방향은 Z축의 양수(+) 방향입니다. 가끔씩은 반대로 돌리고 싶을 때도 있으니 반대로 돌도록 해봅시다. 
    MoveDirection’ 는 Boolean 값을 가진 변수입니다.
    ‘BeginPlay’ 이벤트에서 게임이 시작할때 true / false 를 받아서 액터의 회전 방향을 결정
![alt text](image-44.png)

## 이동하는 액터
    이동하는 액터를 구현하기 위해 블루프린트 클래스를 만들어주겠습니다. 
![alt text](image-45.png)

    사용할 변수는 3개입니다.
    **‘StartLocation’** :  액터가 배치된 기본 위치
    **‘PlatformVelocity’** : 액터가 움직일 방향과 속도
    **‘MoveDistance**’ :  액터가 움직일 거리
![alt text](image-46.png)

    BeginPlay 이벤트에 액터의 기본 위치를 StartLocation 변수에 초기화
![alt text](image-47.png)

    이동은 회전에서 했던것과 거의 같습니다. DeltaSeconds 에 속도를 곱해준 값을 현재 위치에 더해서 ‘Set Actor Location’ 노드에 입력해줍니다.
    이 노드들로 한쪽방향으로 움직이는 것은 구현이 되었습니다.
![alt text](image-48.png)

    ‘Greater Equal’ 연산자를 사용해서 현재 액터의 위치가 이동할 거리(MoveDistance) 보다 크다면
     방향을 반대로 (Negate Vecotr) 바꿔주는 노드들을 추가
![alt text](image-49.png)

![alt text](image-50.png)

## Overlap - Ice
    ‘BP_Ice’ 이름의 블루프린트를 만들어줍니다. 사용한 스테틱 메시는 ‘SM_FloorBase’입니다.
![alt text](image-56.png)

![alt text](image-57.png)

    추가한 컴포넌트는 **‘Box Collision’** 입니다. 적당히 넓게 깔아주시면 됩니다.
    이미지를 보셨다면 눈치채셨겠지만 주황색 콜리전 박스도 스테틱 메시 위로 얇게 펴서 깔아줬습니다.
![alt text](image-58.png)

![alt text](image-59.png)

![alt text](image-60.png)

    첫번째로 작성할 그래프는 **Get Player Pawn > Cast To BP_NBC_Character > 변수로 승격** 입니다. 
    **BeginPlay** 이벤트에 연결 시켜주시면 됩니다.
    이 그래프는 오버랩이벤트에서 작동할 액터를 지정해주기 위해서 입니다.
    모든 액터가 이벤트를 발동시키면 안되겠죠?
![alt text](image-61.png)

![alt text](image-62.png)

    왼쪽 위의 **‘Box Collision’** 컴포너트를 꼭 클릭하시고 **‘OnComponentBeginOverlap’** 이벤트를 만들어줘야합니다. 
    **Overlapped Component (빨강)** 는 오버랩 이벤트가 발동되는 자기 자신입니다.
    **Other Actor, Other Comp (파랑)** 는 오버랩 된 액터, 컴포넌트 입니다.
![alt text](image-63.png)

![alt text](image-64.png)

    첫번째는 ‘Branch’ 노드를 통해 오버랩된 액터를 검사
![alt text](image-65.png)

    저장된 변수에서 ‘Character Movement’ 를 호출하고 지면 마찰 (SetGroundFriction) 은 0, 
    감속 걷기 제동 (SetBrakingDecelerationWalking)은 100을 세팅
![alt text](image-66.png)

    미끄러집니다. 하지만 변경한 계수가 그대로 유지가 되어 빙판을 벗어나도 계속 미끄러집니다.
![alt text](image-67.png)
    
    ‘OnComponentEndOverlap (Box)’ 오버랩 엔드 이벤트로 계수를 원래대로 돌려야합니다. begin 이벤트에서 그대로 복-붙해오시면 됩니다.

## 축을 중심으로 회전하는 액터 + 튕겨내기
    제자리에서 회전하는 액터가 아닌 거리를 두고 축을 중심으로 회전하도록 해보겠습니다. 또 한번, 블루프린트 클래스를 만들어주세요. ‘BP_RotateCar'
![alt text](image-87.png)

    사용한 메시는 **‘SM_PowerCar_02’** 입니다.
    **‘Radius’** : 가상 축까지의 거리 (반지름) - 기본값 700
    **‘Origin**’ : 원래 위치를 저장
    **‘Speed’** : 속도 - 기본값 100
    변수 3개
![alt text](image-88.png)

    첫번째로는 일단 제자리에서 돌려줍니다.
![alt text](image-89.png)

    두번째가 중요합니다. ‘BeginPlay’ 로 런타임이 시작되면 액터 내의 스태틱 메시만 Radius만큼바깥으로 보내줍니다. x축에 radius를 더하기합니다.
![alt text](image-90.png)

    자동차가 원하는대로 축을 기준으로 회전하긴 하는데 캐릭터와 부딪혔을때 제대로 밀어내지 못하는 것 같습니다.
    스태틱 메시에 Hit 이벤트를 활용해서 캐릭터를 강제로 날려보겠습니다.
![alt text](image-91.png)

    스태틱 메시를 클릭한 상태에서 ‘OnComponentHit’ 이벤트 노드를 만들어줍니다.
![alt text](image-92.png)

![alt text](image-93.png)

     Hit 충돌 이벤트도 오버랩이벤트와 캐릭터외에 다른 물체들과 충돌한 경우는 제외하고 싶습니다. 
     아래의 그래프를 BeginPlay의 이벤트에 오버랩이벤트와 똑같이 세팅
![alt text](image-94.png)

    Branch로 캐릭터가 맞는지 확인하고 맞다면, ‘Launch Character’ 노드를 통해 캐릭터를 충돌의 반대방향으로 밀어냅니다. 
    이 때 밀어내는 값은 ‘OnComponentHit’ 이벤트의 Hit 구조체를 핀 분할 한 Hit Normal (정규화된 충돌 벡터) 입니다.
     Hit Normal에 -550을 곱해서 반대방향 먼 곳으로 날려버릴 수 있게 되었습니다.
![alt text](image-95.png)

    Character Movement Component 의 기본 동작 방식때문에 이동 중이 아닌 경우에는 충돌이 제대로 처리되지 않습니다. 이를 해결하기 위해서는 여러가지 트릭을 사용하지만 아쉽게도 이번 섹션에서는 다루지 않습니다.
