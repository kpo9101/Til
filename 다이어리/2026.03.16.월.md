### [Commit 전 주의사항]
```
비주얼 스튜디오를 끄고 커밋해야함.
혹시라도 저장하지 않은 파일이 있다면, 비주얼 스튜디오를 끄면서 저장하겠냐는 창이 나옴.
파일을 저장하지 않은 채로 커밋/푸시하면 어떤 파일은 올라가고 어떤 파일은 그대로 잔류.
습관적으로 모든 파일 저장 및 커밋 전 비주얼 스튜디오 꺼두기.
```

### [주의사항]
```
.gitignore 파일의 위치가 중요함.
반드시 .uproject 파일이 있는 폴더에 함께 위치해야함.
앞으로 작성될 .gitignore 파일 내용은 이를 가정하고 작성되었기 때문.
다른 위치에 위치하게 된다면 제대로 동작하지 않게 됨.
이를 유념한 채로 앞으로의 실습을 진행한다.
```


### 우리가 해볼 상상 훈련들.
```
1. 팀원이 “관련 내용 커밋 푸시 했습니다!”라고 알렸을 때
    내려 받아야하는 상황.
    → 패치/풀만 하면 끝.
2. 내가 작업한 내용을 리모트 레포지토리에 올리려는데,
    아주 운좋게도 작업 시작 전부터 올리려는 시점까지
    아무도 리모트 레포지토리를 건들지 않은 상황.
    → 패치/풀하고, 머지만 하고 커밋 푸시 끝!
3. 내가 작업한 내용을 리모트 레포지토리에 올리려는데,
    이미 팀원이 리모트 레포지토리에 작업 내용을 올린 상황.
    하지만 또 운 좋게 팀원이 올린 작업 내용과 내 작업 내용이 다른 파일.
    → 패치/풀하고, 머지만 하고 커밋 푸시 끝!
4. 내가 작업한 내용을 리모트 레포지토리에 올리려는데,
    이미 팀원이 리모트 레포지토리에 작업 내용을 올린 상황.
    게다가 팀원과 내가 같은 파일을 작업해버려서 충돌남.
    → 패치/풀하고, 머지했을 때 충돌 발생. 해당 팀원과 상의 후 충돌 해결.
```

### 체크리스트
```
  1. 빌드 및 실행 후 게임플레이 잘되는지 확인하기.
  2. 언리얼 에디터 종료 후 소스트리 F5 새로고침.
  3. 변경사항 꼼꼼히 읽어보기.
  4. 머지 직전에 반드시 패치/풀하기.
```
